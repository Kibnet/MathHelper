# SPEC LINTER (Quest Spec)

Для каждого пункта отметить: PASS / PARTIAL / FAIL + комментарий.

## A. Полнота спеки
1. Есть чёткая цель.
2. Описано текущее состояние (AS-IS).
3. Сформулирована одна проблема.
4. Цели дизайна ограничивают решение.
5. Non-Goals заданы явно.

## B. Качество дизайна
6. Есть распределение ответственности.
7. Явно указаны точки интеграции.
8. Бизнес-логика формализована (если есть).
9. Описана обработка ошибок.
10. Учтена производительность (если релевантно).

## C. Безопасность изменений
11. Описаны изменения модели данных.
12. Есть план миграции.
13. Учтена обратная совместимость / откат.

## D. Проверяемость
14. Есть измеримые критерии приёмки.
15. Есть план тестирования.
16. Есть команды для проверки.

## E. Готовность к автономной реализации
17. Есть пошаговый план.
18. Нет блокирующих открытых вопросов.
19. Масштаб задачи соответствует глубине спеки.

## F. Соответствие профилю
20. Выполнены требования выбранного профиля.

### Итог
- ГОТОВО: нет FAIL в разделах A, C, D
- НУЖНА ДОРАБОТКА: есть FAIL
